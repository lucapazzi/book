Part-Whole Statecharts

Theory, Design, Practice

A comprehensive methodology for designing and verifying hierarchical reactive systems

Welcome

This is a complete, self-contained tutorial on Part-Whole Statecharts (PWS)—a formal methodology for building reactive controllers whose correctness emerges from the design itself, rather than being verified after the fact.

The book is organized into eight integrated parts, each building on the previous. The progression moves from philosophical motivation (Introduction and Part I) through formal foundations (Parts II–III) to practical methodology and tools (Parts IV–VI), concluding with synthesis and reflection (Postface).

How to read this book

This material is designed to be read sequentially, from Introduction through Postface. Each part depends on concepts introduced in previous parts. However, you can also:

  • Skip to Part VI if you want immediate hands-on practice with PWSEditor.
  • Focus on Parts II–IV if you are interested in the core design methodology.
  • Start with Part I if you want to understand the problems PWS solves.

This HTML collection was created by synthesizing:

  • Original PDF sources from the PWS project archive

All eight parts share a uniform design system: typography, colour palette, spacing, and interactive elements. This ensures that the book reads as a single, coherent whole, even though each part can also stand alone.

Key Concepts at a Glance

Assembly — A collection of independent component state machines whose behaviour you must coordinate.

Controller — The hierarchical state machine that coordinates assemblies by observing component states and issuing commands.

Configuration Semantics (Sem) — The set of assembly configurations that each controller state is intended to represent.

Exit Zone — An autonomous component evolution that would move the system outside the current controller state's semantics; must be handled by a controller reaction.

Correct-by-Construction — Design methodology where controller correctness emerges incrementally during design, guided by semantic feedback, rather than being verified after completion.

Reactive Space — All assembly configurations reachable through autonomous evolution while the controller remains in a given state.

Compositional Verification — Verification technique where each controller is verified together with its directly controlled assembly; verified subsystems become abstract components for higher levels, avoiding global state explosion.
